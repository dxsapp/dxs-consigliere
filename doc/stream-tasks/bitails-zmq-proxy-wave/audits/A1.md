## Residuals
- The Bitails proxy contract is still provisional vendor surface; only the confirmed topics are implemented in this wave.
- Current config still exposes `connection.zmq.txUrl` and `connection.zmq.blockUrl`; runtime presently requires them to point at the same proxy endpoint when both are set.
- API key handling remains optional/no-auth for this path because Bitails said it is not currently required.
