# A1 Audit

## Scope

Audit the final state of `legacy-convergence-program` after:
- `LC1` capability contract cleanup
- `LC2` raw transaction convergence
- `LC3` validation capability convergence
- `LC4` historical scan truthfulness
- `LC5` broadcast multi-target
- `LC6` dead legacy removal

## Verdict

The program goal is met.

There is now one coherent capability-first model across:
- docs
- runtime wiring
- public validation semantics
- provider/admin wording

## What Is Now Coherent

### Provider posture

- `Bitails` is the default `realtime_ingest` path.
- `JungleBus / GorillaPool` is the preferred `raw_tx_fetch` and default `block_sync` path.
- `WhatsOnChain` remains a REST fallback path rather than a primary raw-tx recommendation.

### Validation authority

- `(D)STAS` truth remains local to `Consigliere`.
- providers are documented and wired as data suppliers, not validation authorities.
- `validation_fetch` is retained as a first-class support capability for lineage dependency acquisition.

### Historical scan truthfulness

- `historical_address_scan` is now explicitly Bitails-backed in v1.
- `historical_token_scan` is now explicitly Bitails-backed rooted-history expansion in v1.
- config and routing no longer advertise fake historical fallbacks.

### Broadcast semantics

- public broadcast now uses multi-target `any_success` semantics.
- attempt truth is persisted and exposed via `Broadcast.Attempts`.
- legacy rebroadcast bypasses have been removed from the remaining long-unconfirmed monitor path.

## Residuals

- `validation_fetch` remains mostly a semantic/config contract today; there is not yet a separate broad dependency-acquisition subsystem that routes every lineage repair action through it.
- historical scans are truthful but still intentionally not generalized beyond Bitails in v1.
- some unrelated compiler warnings remain in the wider solution and were not in scope for this program.

## Acceptance

Accepted as closed because:
- the operator-facing capability model is no longer semantically mixed
- runtime behavior now matches the most important declared product flows
- remaining residuals are honest v1 boundaries, not hidden contradictions
