# A1 Interpreter Follow-Up Audit

## Decision

Oracle demotion is **still blocked**.

Current native interpreter state is strong enough to serve as the primary local parity engine for:
- vendored DSTAS conformance vectors
- vendored protocol-owner chains, including chain-local completion of missing prevouts
- repo-needed forkid sighash variants

But it is not yet justified to demote the deterministic oracle from primary truth anchor to regression/backstop anchor repository-wide.

## Why Demotion Is Still Blocked

The missing proof is not in the current protocol-owner or conformance surface. That part is now materially stronger.

The blocking gap is broader native adoption evidence:
- no explicit native replay proof yet for the full DSTAS master lifecycle surface
- no audit-backed statement yet that broader multisig/lifecycle packs all run through the native interpreter path
- therefore the repository still lacks one final proof layer that local execution truth is comprehensive enough to replace oracle primacy

## What This Wave Did Prove

1. The previous positive-vector blocker was real parity drift and is now removed.
2. Protocol-owner fixtures no longer need partial native parity; missing prevouts can be completed deterministically from chain-local tx graph.
3. The public interpreter surface is now documentable and bounded:
   - resolver contract
   - policy contract
   - trace contract
   - current proven sighash matrix

## Role Classification After This Wave

- native interpreter: `primary local parity engine`
- deterministic oracle: `primary truth anchor still retained`

This is a stronger position than before, but not yet full demotion.

## Required Next Proof Before Demotion

At minimum:
- explicit native interpreter coverage for the remaining broader DSTAS lifecycle packs
- a closeout pass showing those packs are green without oracle-only escapes
