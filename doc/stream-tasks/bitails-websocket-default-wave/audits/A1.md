# Audit A1

## Verdict

`bitails websocket default-on onboarding` is now represented honestly in docs, examples, and provider configuration UX.

## Confirmed

- product docs treat `Bitails websocket` as the first-run realtime path
- `Bitails` API key is described as optional for first-run websocket onboarding
- provider examples no longer imply the key is mandatory for websocket startup
- `/providers` copy now tells operators they can leave the Bitails API key empty on day one
- no document claims a formally documented unlimited Bitails websocket SLA

## Residual

- runtime defaults and source-selection behavior were not changed in this wave
- operators still need to understand paid or higher-limit usage separately from the default-on onboarding story
