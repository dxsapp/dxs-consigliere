# A1 Audit

## Verdict

`raw-tx-provider-recommendation-wave` is acceptable to close.

The product surface now distinguishes clearly between:
- `Bitails` as the recommended realtime provider
- `JungleBus / GorillaPool` as the recommended practical raw transaction fetch path
- `WhatsOnChain` as the simple REST fallback and onboarding provider

## What Changed

- provider policy docs no longer imply that `WhatsOnChain` is the preferred raw transaction path
- Providers page copy now recommends `JungleBus / GorillaPool` for raw tx fetch explicitly
- bounded provider-catalog metadata now exposes the raw-tx recommendation so the UI can render it directly

## What Did Not Change

- realtime defaults did not change
- runtime routing defaults did not change
- `WhatsOnChain` remains in the supported provider contract and remains the default REST recommendation
- no claim is made about a formally documented unlimited JungleBus SLA

## Resulting Product Posture

- newcomers get simpler guidance about which provider solves which problem best
- the ecosystem page reflects actual operational practice more honestly
- the wave stays bounded and does not introduce routing churn
