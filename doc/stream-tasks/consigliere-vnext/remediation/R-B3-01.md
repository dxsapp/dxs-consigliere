# R-B3-01 Address Projection Reorg Request Amplification

## Trigger

- opened from post-rollout recovery benchmark wave `B3`
- problem: `AddressProjectionRebuilder` exhausted Raven session request ceilings on larger synthetic confirmed-block and disconnect-revert bursts

## Root Cause

`AddressProjectionRebuilder` bulk-loaded applied transactions and input UTXOs, but still performed many per-document `LoadAsync` calls for:
- output-side address UTXO projection documents
- address balance projection documents
- revert-time credit/debit dependency lookups

This made larger apply and reorg bursts request-heavy even though all required document identities were already derivable from the batch.

## Fix

- preload output-side address UTXO projection ids in batch context
- preload relevant address balance projection ids in batch context
- store null hits in the batch dictionaries so missing docs do not trigger repeated `LoadAsync` calls
- preload revert dependencies for disconnected-block application batches before iterating per-transaction revert work
- add a targeted integration test for a larger confirmed-block disconnect scenario
