## Quality Review

- Status: `completed`
- Findings:
  - `resolved` `TxLifecycleProjectionRebuilder` originally issued one Raven load per tx observation during replay. `S16` replaced that with page-level bulk loads and an in-session cache, eliminating the session request-ceiling failure in the tx lifecycle benchmark harness.
  - `good` Mixed journal replay remains type-safe: tx lifecycle rebuild ignores foreign observation records while still advancing the shared checkpoint.
  - `good` Out-of-order handling is explicit for the current fact set. Late mempool/drop observations no longer downgrade a confirmed tx, `FirstSeenAt` uses the earliest observed timestamp, and `LastObservedAt` uses the latest observed timestamp.
  - `watch` `TxLifecycleStatus` declares `broadcast_submitted` and `seen_by_source`, but current observation sources only emit `seen_in_mempool`, `seen_in_block`, and `dropped_by_source`. This is acceptable for the current additive endpoint because the response is string-based and consumers only see populated states, but future broadcast/source-liveness work must either emit those statuses or trim the forward-declared enum surface.
