# A1 Swap + Confiscation Audit

## Decision

Oracle demotion is now justified.

## Why

This wave closes the three remaining proof gaps from the prior lifecycle audit:
1. `swap_valid` adds explicit native positive proof for the swap-initiation leg.
2. `confiscate_multisig_valid` adds explicit native positive proof for confiscation under multisig authority policy.
3. `confiscate_multisig_insufficient_signatures_rejected` adds explicit native negative proof for insufficient authority signatures on confiscation.

Those proofs are not isolated in the BSV conformance pack only. The broader DSTAS lifecycle suites now call them explicitly at the relevant swap and multisig authority seams.

## Role Classification After This Wave

- native interpreter: `primary local execution truth engine`
- deterministic oracle: `regression/backstop anchor`

## Residuals

- The deterministic oracle still remains valuable for drift detection and fixture integrity.
- This demotion does not claim exhaustive proof for every future DSTAS variant outside the currently vendored surface.
