# A1 Lifecycle Native Proof Audit

## Decision

Oracle demotion still does **not** reopen after this wave.

The repository now has materially broader native execution proof across DSTAS lifecycle, rooted-history, and multisig-oriented suites, but the proof remains intentionally partial in a few bounded places.

## What This Wave Closed

1. Broader DSTAS full-system suites no longer rely only on synthetic projection state; they now attach explicit native execution proof through vendored conformance vectors.
2. Multisig-oriented lifecycle suites now attach explicit native execution proof through vendored protocol-owner chains and generic confiscation execution proof.
3. Rooted-history/read-surface suites now attach explicit native execution proof for the trusted branch they expose.
4. No protocol-core support change was required; `L4` stayed `not_opened`.

## Why Oracle Demotion Is Still Blocked

The remaining blind spots are now precise:
- no explicit native positive proof yet for the repo's swap-initiation leg; only `swap_cancel_valid` is natively covered in the broader lifecycle suite
- no explicit native proof yet for confiscation under multisig authority policy
- no explicit native negative proof yet for insufficient multisig authority signatures on confiscation or unfreeze paths

These gaps are smaller than before, but they still matter if the oracle role is to be reduced with confidence.

## Role Classification After This Wave

- native interpreter: `primary local parity engine with broader lifecycle proof`
- deterministic oracle: `primary truth anchor still retained`

## Exact Next Proof Needed Before Demotion Reopens

At minimum:
- one native positive proof for swap initiation that matches the repo's lifecycle semantics
- one native positive proof for confiscation under multisig authority policy
- one native negative proof for insufficient multisig authority signatures on an authority action
